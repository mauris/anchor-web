Welcome

Anchor builds on top of the Apache HTTP Server in WampServer and takes advantages of Windows hosts file to create a neat web development experience for you. No more http://localhost/example URLs.
One local Apache HTTP Server instance that serves you PHP, Perl, Python and static HTML+JS applications with ease of configuration and management.
A well designed, simple and familiar user interface to manage multiple websites without the need of copying applications to www or htdocs folder.
You can be assured that the settings for the VirtualHosts and hosts file are backed up and secured properly. The domains are hosted locally and cannot be accessed from outside.

The Interface

The user interface is simple and easy to use for any developer. As you only need to use Anchor to manage your hosts, you can close Anchor and the hosts will still work.

The following explains the Anchor's user interface shown on the left.

  • 1 Settings button - Shows options to configure Anchor.
  • 2 On/Off switch - Turns the Anchor services on or off.
  • 3 Add Anchor button - Shows the form to add a new domain name.
  • 4 Anchor list - Lists the domains hosted and their options.
  • 5 Open button - Opens the anchor in the default browser.
  • 6 Browse button - Opens the folder in Windows Explorer.
  • 7 Delete button - Confirms and delete the anchor host.

Behind the Scenes

Anchor does three very simple things:

  1. Maintain a list of hosts and their directory pathnames
  2. Write these hosts to a Apache HTTP Server configuration file called anchor.conf
  3. Update the Windows hosts file with the list of domain names

In an event that your hosts file gets corrupted or your entries becomes missing (which shouldn't happen), you can find the original pre-Anchor hosts file as hosts.orig.
